# PosterNung (posternung)

A Flutter app for discovering and collecting rare, authenticated original movie posters.

[![CI](https://github.com/Chanothai/posternung/actions/workflows/ci.yml/badge.svg)](https://github.com/Chanothai/posternung/actions/workflows/ci.yml)

## Tech Stack

| Concern           | Choice                                                                    |
| ------------------ | -------------------------------------------------------------------------- |
| Framework          | Flutter (stable channel)                                                   |
| State management   | [Riverpod](https://pub.dev/packages/flutter_riverpod)                      |
| Architecture       | Clean Architecture, feature-first (see [CLAUDE.md](CLAUDE.md))             |
| Backend            | Firebase Auth (email/password, Google, Apple)                              |
| Fonts              | Google Fonts — Kanit throughout (Thai-glyph coverage)                      |
| Testing            | `flutter_test`, [`mocktail`](https://pub.dev/packages/mocktail)            |
| CI/CD              | GitHub Actions (`.github/workflows/ci.yml`)                                |

Architecture rules, folder conventions, and code patterns are documented in **[CLAUDE.md](CLAUDE.md)** — read that before adding a feature. Commit message and branching conventions are documented in **[CONTRIBUTING.md](CONTRIBUTING.md)**.

## Project Structure

```
posternung/
  lib/
    core/                  # cross-feature code — see lib/core/CLAUDE.md
      theme/                # AppColors, AppTextStyles
      design_system/        # AppSpacing, AppRadius, AppDimens
      strings/               # AppStrings — centralized UI copy
      assets/                # AppImages — centralized asset paths
      config/                # build-environment resolution (SIT/UAT/Production)
      error/                 # shared domain exception types
      widgets/                # cross-feature widgets (AppGradientBackground, ...)
    features/
      onboarding/            # 3-screen intro flow (presentation-only)
      auth/                   # login/register — full domain/data/presentation chain
      home/                    # post-login poster marketplace
    firebase_options_{sit,uat,production}.dart  # generated by flutterfire configure
    main.dart                # app bootstrap: Firebase, ProviderScope, root MaterialApp
  test/                     # mirrors lib/ — see CLAUDE.md testing conventions
  assets/images/            # icons/art pulled from Figma
  android/ ios/ web/ macos/ linux/ windows/   # Flutter platform shells
  docs/                     # environments, git workflow, social login setup
```

Each feature has its own scoped `CLAUDE.md` (`lib/features/<name>/CLAUDE.md`) documenting its current file structure and any feature-specific notes.

## Features in Development

| Feature | Status | Notes |
| --- | --- | --- |
| **Onboarding** | ✅ Complete | A 3-screen pitch of the app's value — own a piece of cinema history, 100% authenticated originals, and limited one-of-a-kind stock. Users swipe or tap Next through the pages, can Skip on any page, then hand off to Login. |
| **Auth** | 🟢 Email/password + Google | Sign in or register to reach the collection; the app shows Login when signed out and Home when signed in. Apple sign-in is built but hidden pending native entitlements, and social sign-in is mobile-only. |
| **Home** | 🟡 UI complete, mock data | The poster marketplace users land on after login — browse featured collections, ending-soon deals, and the full catalog, then sign out from the Profile tab. Runs on placeholder catalog data today; search, wishlist, cart, and "View all" show a "coming soon" message until the backend lands. |

## Build Environments

The app builds as three environments — **SIT**, **UAT**, **Production** — via native Android flavors and iOS schemes, installable side by side on one device. See [`docs/environments-setup.md`](docs/environments-setup.md).

## Getting Started

```bash
flutter pub get
flutter run --flavor sit
```

Firebase is already configured for each environment. To reconfigure or add a platform:

```bash
flutterfire configure --project=posternung
```

## Testing

```bash
flutter test              # unit + widget tests
flutter test --coverage   # with lcov coverage report at coverage/lcov.info
```

`test/` mirrors `lib/` — usecases and repositories are unit-tested with `mocktail`, widgets are tested via `ProviderScope` overrides so no test touches real Firebase. See **[CLAUDE.md § Testing](CLAUDE.md#testing)**.

## CI/CD

`.github/workflows/ci.yml` runs on every push to `main` and every pull request:

1. **format** — `dart format --set-exit-if-changed`
2. **analyze** — `flutter analyze`
3. **test** — `flutter test --coverage`, coverage report uploaded as a build artifact
4. **build** (after format/analyze/test pass) — Android APK, iOS (no codesign), Web

All jobs are cached (Flutter SDK + pub packages) and cancel superseded runs on the same branch/PR automatically.
