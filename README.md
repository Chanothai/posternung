# Cinevault (posternung)

A Flutter app for discovering and collecting rare, authenticated original movie posters.

[![CI](https://github.com/Chanothai/posternung/actions/workflows/ci.yml/badge.svg)](https://github.com/Chanothai/posternung/actions/workflows/ci.yml)

## Tech Stack

| Concern            | Choice                                            |
| ------------------ | -------------------------------------------------- |
| Framework           | Flutter (stable channel)                           |
| State management    | [Riverpod](https://pub.dev/packages/flutter_riverpod) |
| Architecture         | Clean Architecture, feature-first (see [CLAUDE.md](CLAUDE.md)) |
| Backend              | Firebase (Auth today; more services as features need them) |
| Fonts                | Google Fonts (Lora, Libre Baskerville)             |
| Testing              | `flutter_test`, [`mocktail`](https://pub.dev/packages/mocktail) |
| CI/CD                | GitHub Actions (`.github/workflows/ci.yml`)         |

Architecture rules, folder conventions, and code patterns are documented in **[CLAUDE.md](CLAUDE.md)** — read that before adding a feature. Commit message and branching conventions are documented in **[CONTRIBUTING.md](CONTRIBUTING.md)**.

## Project Structure

```
posternung/
  lib/
    core/
      theme/                        # design tokens: AppColors, AppTextStyles (from Figma)
    features/
      onboarding/
        presentation/
          providers/                # OnboardingController (Riverpod Notifier) — page index state
          screens/                  # OnboardingFirstScreen
    auth/                           # Firebase Auth: AuthGate (auth-state gate), LoginPage (email/password)
    home/                           # placeholder post-login screen (to be replaced by the real app shell)
    firebase_options.dart           # generated by `flutterfire configure` — safe to commit (public client config)
    main.dart                       # app bootstrap: Firebase.initializeApp, ProviderScope, root MaterialApp
  test/
    widget_test.dart                # mirrors lib/ — see CLAUDE.md testing conventions
  assets/
    images/                         # icons/art pulled from Figma
  android/ ios/ web/ macos/ linux/ windows/   # Flutter platform shells
  .github/workflows/ci.yml          # CI: format, analyze, test, build (android/ios/web)
```

`auth/` and `home/` predate the Clean Architecture convention in CLAUDE.md and haven't been migrated into the feature-first layout yet — see the "Features in development" note below.

## Features in Development

| Feature | Status | Notes |
| --- | --- | --- |
| **Onboarding** | 🟡 In progress | First of 3 screens built (`OnboardingFirstScreen`, Figma node `6:2`). Screens 2–3 not yet designed/built. Progress-dot state is driven by `onboardingControllerProvider`. Hero background is a placeholder gradient — the Figma source frame's poster-collage fill hasn't been generated yet. |
| **Authentication** | 🟡 In progress | Email/password sign-in and sign-up via Firebase Auth (`AuthGate` + `LoginPage`). Firebase project (`posternung`) is configured for Android/iOS/Web, but the **Email/Password sign-in provider still needs to be enabled in the Firebase Console** (Authentication → Sign-in method) before login/signup will work at runtime. Not yet reorganized into `features/auth/{data,domain,presentation}`. |
| **Home / main app shell** | ⚪ Placeholder | `home/home_page.dart` is the stock Flutter counter demo, shown after a successful login. Needs to be replaced with the real post-onboarding app shell. |

Onboarding is currently the app's entry point (`main.dart` → `OnboardingFirstScreen`); tapping **Skip** or reaching the last progress dot and tapping **Next** pushes into `AuthGate`, which shows `LoginPage` when signed out or the home shell when signed in.

## Getting Started

```bash
flutter pub get
flutter run
```

Firebase is already configured for this project (`lib/firebase_options.dart`, `android/app/google-services.json`, `ios/Runner/GoogleService-Info.plist`). If you need to reconfigure or add a platform:

```bash
flutterfire configure --project=posternung
```

## CI/CD

`.github/workflows/ci.yml` runs on every push to `main` and every pull request:

1. **format** — `dart format --set-exit-if-changed`
2. **analyze** — `flutter analyze`
3. **test** — `flutter test --coverage`, coverage report uploaded as a build artifact
4. **build** (fans out after format/analyze/test pass) — `flutter build apk --debug`, `flutter build ios --no-codesign`, `flutter build web`

All jobs are cached (Flutter SDK + pub packages) and cancel superseded runs on the same branch/PR automatically.
